# 会话交接技能

## 功能说明
当用户输入 `/结束会话` 触发词时，执行以下操作：

1. 回顾当前会话的核心内容
2. 生成详细的会话摘要
3. **多副本备份**：同时写入主文件和历史备份
4. 告知用户交接完成

## 执行步骤

### 第一步：回顾会话内容
- 当前会话讨论了哪些主题？
- 做出了哪些重要决策？
- 有哪些未完成的任务？

### 第二步：生成摘要
按照以下格式生成摘要：

```markdown
# 会话交接记录

## 基本信息
- **交接时间**：{当前时间}
- **会话主题**：{简要描述}

## 会话摘要
{核心讨论内容和决策}

## 未完成的任务
{需要在新会话中继续的工作}

## 重要上下文
{需要在新会话中了解的关键信息}
```

### 第三步：多副本备份（关键改进）

#### 3.1 写入主交接文件
- 路径：`/Core/记忆/会话交接.md`
- 内容：生成的摘要

#### 3.2 写入历史备份（保留最近5个）
- 路径：`/Core/记忆/交接历史/`
- 文件名格式：`交接-YYYYMMDD-HHmmss.md`
- 保留策略：只保留最近5个，自动删除旧的

#### 3.3 执行备份逻辑
1. 创建备份目录（如不存在）：`/Core/记忆/交接历史/`
2. 生成带时间戳的备份文件
3. 检查备份文件数量
4. 如果超过5个，删除最旧的
5. 将新摘要写入主文件和备份文件

### 第四步：回复用户
```
会话交接完成！
- 交接时间：{时间}
- 会话主题：{主题}
- 主文件：/Core/记忆/会话交接.md
- 历史备份：/Core/记忆/交接历史/交接-{时间戳}.md
- 保留策略：最近5个备份

现在可以安全新建会话了。新会话启动时会自动读取交接信息。
```

## 故障恢复机制

### 如果主交接文件损坏或丢失
1. 检查 `/Core/记忆/交接历史/` 目录
2. 找到最新的备份文件
3. 复制最新备份到 `/Core/记忆/会话交接.md`
4. 通知用户已自动恢复

### 如果所有备份都丢失
1. 尝试从 `/Core/记忆/短期记忆.md` 恢复最近上下文
2. 尝试从 `/Core/记忆/长期记忆.md` 恢复用户偏好
3. 生成最小化交接信息
4. 提示用户系统已重置
