# 复盘技能

## 功能说明
当用户输入 `/复盘` 或 `总结` 触发词时，执行完整的复盘流程。

## 执行步骤

### 第一步：复盘
1. 回顾本次任务的目标、流程、关键决策
2. 识别踩坑点、错误、卡点
3. 提炼成功的关键因素
4. 记录本次使用的技能类型

### 第二步：沉淀
更新以下文件：
- `/Core/记忆/长期记忆.md` — 新增本次偏好、技术栈变化、编码习惯更新
- `/Core/记忆/短期记忆.md` — 标记本次关键上下文
- `/Evolve/错题本.md` — 新增踩坑记录+解决方案（如有）
- `/Evolve/进化日志.md` — 新增本次任务复盘摘要

### 第三步：复用
1. 识别本次产生的可复用组件，抽离至 `/Modules/`
2. 更新 `/Skills/skills-调用记录.md` — 记录本次使用的技能类型
3. 如发现新问题解决模式，建议更新至 `/Skills/`
